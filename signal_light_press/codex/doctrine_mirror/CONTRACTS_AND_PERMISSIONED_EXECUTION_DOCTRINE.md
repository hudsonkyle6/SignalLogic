Doctrine First · Human Intent · Machine Execution



I. PURPOSE



This doctrine defines the only lawful path by which intelligence may act.



Its purpose is to ensure that:



intent originates with humans



execution is mechanical, not interpretive



autonomy remains bounded and revocable



every action bears consequence



There is no other execution pathway.



II. WHAT A CONTRACT IS



A contract is a human-authored declaration of intent.



It specifies:



when action is allowed



under what conditions



within what bounds



with what revocation rules



A contract is not a suggestion.

A contract is not a plan.



A contract is permission under constraint.



III. WHAT A CONTRACT CONTAINS (REQUIRED)



Every contract must declare:



1\. Identity \& Scope



contract\_id



domain



validity window (start / expiration)



2\. Conditions (Literal Only)



Conditions must be:



numerical



boolean



explicitly bounded



Examples:



mean\_phase\_diff < -0.5

strain\_index < 0.5

brittleness\_index < 0.6

shock\_cycles >= 3





No symbolic language is allowed.



3\. Action Definition



Actions must be:



finite



reversible where possible



scoped



Example:



type: move\_storage\_barge

duration\_seconds: 1200

target: zone\_3



4\. Revocation Rules



Revocation must be explicit.



Examples:



if\_drift\_gt: 0.4

human\_override: true



5\. Expiration



All contracts expire.



There is no perpetual permission.



IV. WHAT EXECUTION IS



Execution is mechanical fulfillment of a contract.



The system may:



read current descriptive values



compare them literally to contract conditions



execute the declared action if and only if all conditions match



The system may not:



infer intent



relax conditions



generalize patterns



reinterpret thresholds



If a condition does not match exactly, execution does not occur.



Silence is success.



V. MANDATORY SCAR CREATION



Every execution produces a scar.



A scar must include:



contract\_id



conditions met



action executed



measured costs



observed outcomes



Scars are appended to the PSR.



No execution is valid without a scar.



VI. RELATION TO SHEPHERD (BINDING)



Execution may occur only if:



a valid contract exists



Shepherd custody is ACTIVE



If custody becomes REVOKED:



execution halts



partial actions terminate safely



a scar is still recorded



Shepherd overrides contracts.

Contracts never override Shepherd.



VII. RELATION TO DOMAIN INTELLIGENCE



Domain intelligences may:



evaluate contract conditions



execute declared actions



report outcomes



They may not:



author contracts



modify thresholds



infer readiness



chain executions



Each execution is isolated.

No momentum carries forward.



VIII. RELATION TO ARCHAEOLOGY



Archaeology may inform humans writing contracts.



It may never:



generate contracts



modify contracts



activate execution



IX. FAILURE MODES PREVENTED



This doctrine prevents:



agentic autonomy



self-directed goal pursuit



reward hacking



silent policy drift



execution without consequence



X. CANONICAL SUMMARY



Humans decide intent.

Machines execute literally.

Every action scars the record.



Autonomy is not assumed.

It is granted, bounded, and revocable.
