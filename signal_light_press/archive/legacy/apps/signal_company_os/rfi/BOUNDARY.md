# RFI — Resonant Field Intelligence Boundary

Status: Active Descriptive Intelligence  
Authority: None  
Execution: Permitted (bounded, read-only outputs)  
Role: Ecological signal reduction and coherence description

## Description

RFI (Resonant Field Intelligence) is a bounded ecological intelligence layer.
It reduces observational signals and approved ecological ML outputs into
descriptive summaries suitable for coherence gating.

RFI does not decide.
RFI does not act.
RFI does not predict outcomes.
RFI does not train models.

It exists to describe field state, uncertainty, and alignment — nothing more.

## Relationship to Lighthouse

- Lighthouse explores and trains ecological ML candidates
- RFI may incorporate *approved, frozen* ML components
- No training, tuning, or optimization occurs in RFI
- All ML use in RFI must be bounded, cycle-aware, and inspectable

## Outputs

RFI may output:
- Coherence metrics
- Missingness / void ratios
- Bounded state descriptors
- Confidence intervals or uncertainty measures

All outputs must:
- Be finite
- Be bounded
- Be explicit about missing data
- Default to silence on ambiguity

## Prohibitions

RFI must never:
- Trigger actions
- Override the Gate
- Write into Kernel or Rhythm OS
- Import Lighthouse dynamically
- Perform live learning or optimization
