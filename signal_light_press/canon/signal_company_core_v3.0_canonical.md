## CAN BUS ARCHITECTURE
################################################################################

### 1. The Bus (Shared Backbone)

The bus is the union of:
- Helix Sync (coherence arbitration)
- Dark Field (append-only persistence)

Properties:
- Append-only
- Message-based
- No mutation of past records
- Priority determined by coherence, not command

### 2. Nodes (Distributed Helmsmen)

Each node observes locally and broadcasts when coherent.

Examples:
- Power Helmsman
- Maritime Helmsman
- Logistics Helmsman
- Core Self Helmsman
- Oracle Node
- Antifragile Node
- Lighthouse Node

Nodes NEVER authorize action.
Nodes NEVER silence other nodes.
Nodes ONLY publish packets.

### 3. Arbitration (Helix Sync)

Helix Sync:
- Measures phase lock
- Measures amplitude resonance
- Measures frequency entrainment
- Measures decay coherence

Outputs:
- Global coherence score [0.0–1.0]
- Packet priority ranking

Helix Sync does NOT think.
Helix Sync does NOT decide.
Helix Sync arbitrates signal priority only.

### 4. Silence (Correct Definition)

Silence is:
- Absence of actuation
- Absence of mandate
- Absence of execution

Silence is NOT:
- Absence of observation
- Absence of modeling
- Absence of reasoning
- Obedience
- Moral restraint

The bus is always alive.
