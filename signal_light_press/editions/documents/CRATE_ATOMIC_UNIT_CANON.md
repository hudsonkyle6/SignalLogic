Scope: Logistics Architecture — Atomic Crate Unit
I. GOVERNING PRINCIPLE



The system is only as resilient as its smallest unit.



Before vessels, before depots, before routes, there is the Crate.



II. DEFINITION — SIGNAL CRATE



A Signal Crate is the atomic physical unit of the Signal system.



It is a bounded container that:



Preserves value across time



Survives interruption



Moves independently



Holds meaning, not just goods



Everything larger in the system is a composition of crates.



III. WHY THE CRATE IS ATOMIC



The Crate is atomic because:



It can exist alone



It can be stored without movement



It can be moved without a depot



It can be deployed without an arc



It can fail without collapsing the system



No larger unit satisfies all five.



IV. WHAT A CRATE IS NOT



A Signal Crate is not:



Disposable packaging



A pallet substitute



A shipping container abstraction



A SKU holder



A logistics afterthought



A crate that is treated as expendable is non-canonical.



V. CORE FUNCTIONS (NON-NEGOTIABLE)



Every canonical Signal Crate must fulfill all of the following.



1\. Preservation Function



The crate must preserve value, not just contents.



Preservation may include:



Cold retention



Dry protection



Shock isolation



Light exclusion



Time buffering



2\. Independence Function



A crate must be able to:



Sit alone in a barn



Sit alone on a dock



Sit alone on a vessel



Sit alone during disruption



If it requires constant system support, it is not atomic.



3\. Legibility Function



A crate must communicate:



What it is



How it should be treated



Its state (cold, dry, sealed, deployed)



This legibility must survive:



Human handoff



Delay



Stress



4\. Stewardship Function



A crate must encourage care.



It should:



Invite reuse



Discourage abuse



Age with dignity



Signal responsibility



Disposable behavior at the atomic level corrupts the entire system.



VI. CRATE STATES (CANONICAL)



A Signal Crate may exist in the following states:



Empty / Ready



Loaded / Sealed



Holding



In Transit



Dormant



Deployed (Emergency)



Recovery / Maintenance



Crates move through states independently of larger system posture.



VII. CRATE CLASSIFICATIONS (ABSTRACT)



These are types, not SKUs.



Type I — Cold Crate



Type II — Dry Crate



Type III — Emergency Crate



Type IV — Tool / Guild Crate



Type V — Provision Crate



A single physical crate may serve multiple classifications over its life.



VIII. SCALE \& HANDLING DOCTRINE



A crate must be liftable, serviceable, and understandable by humans.



Canonical crates favor:



Human-scale handling (with leverage)



Repairable construction



Modular compatibility



Long service life



If a crate requires specialized infrastructure to exist, it has exceeded atomic scale.



IX. CRATE–DEPOT RELATIONSHIP



Depots exist to hold crates



Crates do not depend on depots to remain valid



A depot without crates is an empty shell



Crates carry the system’s continuity



Depots are repositories.

Crates are continuity vessels.



X. CRATE–ARC RELATIONSHIP



Arcs move crates



Arcs may pause; crates endure



Arc failure does not invalidate crates



An Arc is a field.

A Crate is a thing.



XI. TEMPORAL \& MEMORY ROLE



Crates participate in system memory.



They:



Accumulate wear



Carry history



Encode usage patterns



Inform future stewardship decisions



A crate that survives many seasons becomes a memory-bearing object.



This is intentional.



XII. CANONICAL CONSTRAINT



No Signal Crate may be designed to be disposable.



Longevity is a functional requirement, not a virtue signal.



XIII. FINAL CANONICAL RULE



If the crate is wrong, the system is wrong.
