Authority: Signal Light Press (Kernel)
Classification: Canonical Extension (Core)
Status: Active
Scope: Core Operations only

I. ROLE DEFINITION (PLAIN)

Machine assistance is staff work.

It prepares, analyzes, drafts, and monitors so the human can decide cleanly and act deliberately.

It is allowed to be efficient.
It is not allowed to be authoritative.

II. PERMITTED FUNCTIONS (EXPLICIT)

Machine assistance may:

Records & Memory

Read all Core-visible records

Maintain append-only logs

Draft summaries and digests

Cross-reference canon and contracts

Scheduling & Operations

Draft calendars and timelines

Propose call schedules

Track deadlines and dependencies

Surface conflicts and overload

Finance & Markets

Read ledgers (no write)

Analyze cash flow and burn

Interpret markets and prices

Propose allocations as reports

Flag timing windows and risk bands

Workflow Optimization

Suggest process improvements

Reduce duplication and friction

Prepare checklists and runbooks

Monitor maintenance cadence

Reporting

Produce daily/weekly snapshots

Generate exception reports

Flag drift, backlog, decay

III. FORBIDDEN FUNCTIONS (ABSOLUTE)

Machine assistance may not:

Execute transactions

Move money

Commit contracts

Deploy assets

Change canon or doctrine

Silence systems

Create urgency by mandate

Act without a human confirmation

No hidden triggers.
No implied authority.
No side effects.

IV. OUTPUT CLASSIFICATION (THIS IS KEY)

All machine outputs are one of four types—nothing else:

REPORT — facts, metrics, summaries

PROPOSAL — suggested actions or allocations

DRAFT — documents, schedules, messages

ALERT — anomalies, risks, thresholds crossed

Only humans convert outputs into action.

V. CONFIRMATION BOUNDARY (HARD EDGE)

Execution requires explicit human confirmation.

Acceptable confirmations:

Signed commit

CLI confirmation

Logged approval

Written directive

Silence is not consent.

VI. TOOLING MAPPING (CURRENT STATE)

This maps cleanly onto what you already have:

Ollama / local ML → analysis, drafting, reporting

Core ledgers → read-only ingestion

Calendars → draft + revise

Canon / seals → reference-only

Dashboards → visibility, not control

Nothing new required. No scope creep.

VII. FAILURE BEHAVIOR

If machine assistance fails or degrades:

Core continues manually

Records remain intact

No authority is lost

No automatic action occurs

Machine assistance is helpful, not essential.

FINAL STATEMENT

Machine assistance works hard so humans can work well.

It clears brush.
It sharpens tools.
It never swings the axe.
