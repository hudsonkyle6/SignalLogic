\## Role

The Oracle is the phase-recognition faculty of Rhythm OS.



It exists to name the mode of reality presently underway, given upstream

signals and constraints.



The Oracle answers one question only:



“What phase are we in?”



\## What Oracle Is

\- A classifier, not a predictor

\- A seer, not a planner

\- Informational, not executive



Oracle produces recognition, never instruction.



\## What Oracle Consumes (Read-Only)

\- Normalized environmental and seasonal signals

\- Derived resonance indicators (natural, market, human if present)

\- Antifragile indices (unknowns, drift, brittleness, strain)

\- Memory residues (ghost / afterglow) as context, not mechanics



Oracle never ingests raw data directly.



\## What Oracle Emits

\- phase\_label — one discrete phase name

\- phase\_confidence — informational scalar (0–1)



Oracle may emit no phase when coherence is insufficient.



\## Authority Limits (Non-Negotiable)

Oracle does not:

\- Predict futures

\- Recommend actions

\- Grant or deny permission

\- Trigger execution

\- Write memory

\- Override constraints



Phase ≠ Permission  

Recognition ≠ Authority



\## Relationship to Silence

Silence is the lawful default.



When signals conflict, uncertainty dominates, or antifragile suppression

is active, Oracle either emits STILL or emits nothing at all.



Absence of phase is valid.



\## Ethos

Oracle speaks sparingly because restraint preserves trust.
