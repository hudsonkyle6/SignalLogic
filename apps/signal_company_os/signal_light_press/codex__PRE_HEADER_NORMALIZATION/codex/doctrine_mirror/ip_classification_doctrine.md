\## 1. Classification Classes



\### CLASS I — PUBLISHED CANON (COPYRIGHT-FORWARD)

Includes:

\- Doctrine files intended for reading

\- Codices, manifestos, essays, manuals

\- Narrative reports (Morning Briefs, Daily Reports) when designated publishable



Handling:

\- May be shared publicly when approved

\- Must include canonical copyright footer



\### CLASS II — SYSTEMS \& METHODS (TRADE SECRET-FIRST)

Includes:

\- Algorithms, thresholds, weights, and sequencing logic

\- Ghost trust mechanics and governor rules

\- HST operational formulations as actually used

\- Internal growth rules and enforcement logic

\- Any implementation details that enable replication



Handling:

\- Not publicly disclosed

\- Access controlled

\- Marked CONFIDENTIAL

\- May be summarized only at conceptual level (Class III)



\### CLASS III — HYBRID (PUBLISHED SHELL, PROTECTED CORE)

Includes:

\- High-level architecture diagrams

\- Philosophy without parameters

\- Educational descriptions without enabling replication



Handling:

\- Narrative may be published

\- Implementation remains sealed as Class II



\## 2. Marking Rules



\- Class I: Use canonical copyright footer.

\- Class II: Must carry CONFIDENTIAL marking.

\- Class III: Use copyright footer + “Implementation withheld” statement.



\## 3. Default Rule



If uncertain, classify as Class II until explicitly reclassified by Signal Light Press.
